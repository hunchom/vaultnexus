# Zettelkasten vs Folders

The recurring religious war in notes-tool communities is whether to organise by hierarchical folders or by a flat-graph Zettelkasten. I have run both extremes and converged on a middle position that I think is more honest about the trade-off than either pole.

## The folder pole

Folders are a system of pre-commitment. You decide, at file-creation time, which category the note belongs to. The cost is that real ideas frequently belong to more than one category, and you have to either duplicate, symlink, or accept that the filing is partial. The benefit is browsability — you can find things by walking the tree even when you have forgotten what they are called.

Folders work well when the categories are stable and exclusive. They work badly when your thinking spans categories or when the categories themselves are still in flux.

## The Zettelkasten pole

A pure Zettelkasten is a flat collection of atomic notes ([[atomic-notes-principle]]) connected by explicit links. There is no folder hierarchy; there is only the link graph. The cost is that the absence of pre-commitment makes browsing impossible — you can only navigate by following links you remember. The benefit is that the graph can represent multi-category membership natively, and emergent structure shows up in the link patterns rather than being imposed up front.

Pure Zettelkasten works well when you have high-quality link discipline and never need to browse. It works badly otherwise.

## The middle position I actually run

I use shallow folders — exactly one level deep in most cases — that approximate topic-level categories without claiming to be a strict ontology. Inside each folder, the structure is link-driven and the folder is more like a community label than a strict container. The folder-as-soft-category pattern catches the browse case (I can walk the top-level folders) without forcing the pre-commitment cost.

The links carry the real organisational weight. Folders are a finding aid; the graph is the system.

## When to break the middle position

If a note is genuinely cross-cutting — it serves three topic clusters equally — I put it in whichever folder has the most outbound links to it and let the link graph carry the rest. The folder placement is a 60/40 hint, not a 100% category claim.
