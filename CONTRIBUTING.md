# Contributing

Thanks for the interest. VaultNexus is a small Node 22 + TypeScript codebase with two halves: a daemon (long-running indexer + HTTP + MCP server) and an Obsidian plugin (sidebar search + settings tab). Both ship in the same monorepo.

## Setup

```bash
git clone https://github.com/hunchom/vaultnexus.git
cd vaultnexus
pnpm install
pnpm run build
pnpm test
```

The Obsidian plugin uses workspace-isolated deps:

```bash
cd obsidian-plugin
pnpm install --ignore-workspace
node esbuild.config.mjs
```

## Dev loop

```bash
# Daemon w/ fake embedder against an empty vault dir
VAULTNEXUS_VAULT=/tmp/empty-vault node dist/daemon/main.js

# Plugin watch (rebuilds main.js on save)
cd obsidian-plugin && node esbuild.config.mjs --watch
```

For the plugin to reload, toggle it off + on in Obsidian's Community plugins panel.

## House style

- **No Claude / Anthropic attribution** in committed files. The project is incognito by policy. No `Co-Authored-By`, no `🤖 Generated by`, no `noreply@anthropic.com`.
- **No secrets in repo.** API keys live in env vars + `.gitignored` files only. The `~/.zshrc.secrets` pattern is recommended.
- **Comments are minimal.** Don't write essays — name things well and let the code speak. The only acceptable inline comment is one that explains a non-obvious WHY (a workaround, an invariant, a surprising constraint).
- **Tests first when fixing a regression.** Add the failing case to `test/`, then fix.
- **One concern per commit.** Conventional Commits: `feat:`, `fix:`, `style:`, `docs:`, `ci:`, `chore:`.

## Tests

```bash
pnpm test                                                # full suite, ~10s
pnpm test test/daemon/                                   # just daemon
pnpm test test/daemon/http-configure-chat.test.ts        # one file
PATH="/opt/homebrew/opt/node@22/bin:$PATH" pnpm test     # if your system node is old
```

The `main.integration.test.ts` suite spawns a real daemon and needs port `:38473` free. Kill any running daemon before running it locally.

## Eval harness

To measure retrieval quality against the seeded paraphrase corpus:

```bash
export VAULTNEXUS_EMBED_URL="https://api.voyageai.com/v1"
export VAULTNEXUS_EMBED_KEY="pa-..."
export VAULTNEXUS_EMBED_MODEL="voyage-3-large"
pnpm exec tsx src/eval/seeded-run.ts
```

You should see something like `queries=26 recall@1=0.92+ MRR=0.95+`.

## Releasing

```bash
git tag -a vX.Y.Z -m "vX.Y.Z — short summary"
git push origin vX.Y.Z
gh release create vX.Y.Z --title "..." --notes "..."
```

CI runs on every push to `master` and every PR — both `ubuntu-latest` and `macos-latest` under Node 22.
